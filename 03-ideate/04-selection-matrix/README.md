# 04 - Selection Matrix

**Purpose:**
This folder contains the final, objective evaluation that concludes the Ideation phase.

**Contents:**
- `evaluation.md`: A scored matrix grading the final concepts against strict project constraints.

**Methodology:**
To prevent subjective bias from choosing the "coolest" but least practical idea, the Selection Matrix forces the team to score each fleshed-out concept (from `03-concept-notes`) objectively from 1 to 5 across specific constraints defined in the `02-define` phase. 

For this project, the critical constraints were: Cost (< ₹4,000), Safety, Modularity, and Non-Disruptive nature. The highest-scoring concepts from this matrix dictate exactly what the team will build in the `04-prototype` phase.
